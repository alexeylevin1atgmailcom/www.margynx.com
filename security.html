Security

This page summarizes security practices and supports Amazon’s requirement for appropriate security measures and a valid HTTPS site. [oai_citation:14‡Amazon Developer Documentation](https://developer-docs.amazon.com/sp-api/docs/website-guidelines)

Security / Support: support@margynx.com
Privacy: privacy@margynx.com

1) Website security

  • HTTPS: margynx.com is served over HTTPS with a valid SSL certificate. [oai_citation:15‡Amazon Developer Documentation](https://developer-docs.amazon.com/sp-api/docs/website-guidelines)
  • No excessive popups/ads/redirects: the website is designed to be simple and readable. [oai_citation:16‡Amazon Developer Documentation](https://developer-docs.amazon.com/sp-api/docs/website-guidelines)

2) Application security controls

  • Least privilege: access is limited to what is required to operate the service.
  • Data isolation: seller data is logically segregated per account.
  • Encryption: data is protected in transit and at rest where stored.
  • Audit and monitoring: operational monitoring and logging are used for security and reliability.
  • Credential handling: secrets/tokens are stored securely and access is restricted.

3) Incident response & support

If you suspect a security issue, contact support@margynx.com. For privacy-related matters, contact privacy@margynx.com.

Margynx and Agentix are intended to comply with Amazon’s Acceptable Use Policy, Data Protection Policy, and Solution Provider Agreement. [oai_citation:17‡Amazon Developer Documentation](https://developer-docs.amazon.com/sp-api/docs/website-guidelines)
