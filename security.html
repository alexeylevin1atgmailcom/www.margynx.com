Security

Security practices

Public overview of how the service is designed to protect customer data and account access.

1. Security posture

Margynx is designed to operate with appropriate administrative, technical, and organizational safeguards proportionate to the service, the sensitivity of the data involved, and applicable platform requirements.

2. Core controls

  • HTTPS and encryption in transit: public pages and service traffic are intended to be served over HTTPS/TLS.
  • Access controls: access to systems and data should be restricted on a least-privilege and need-to-know basis.
  • Credential protection: secrets, tokens, and credentials should be stored using secure handling practices and not exposed in code or public repositories.
  • Logical segregation: customer data should be segregated by account or tenant context within the application architecture.
  • Monitoring and logging: service and security events should be logged and monitored to support troubleshooting, detection, and investigation.

3. Operational practices

  • security updates and patches should be applied in a risk-based and timely manner;
  • production access should be limited and controlled;
  • infrastructure and application changes should follow a managed deployment process;
  • backups and recovery procedures should be maintained where appropriate to the service design.

4. Seller-authorized access

Agentix Seller Cockpit is designed to access Amazon-related data only through seller-authorized workflows and only within the scope of permissions granted. Users may revoke authorizations through the relevant provider flows where supported.

5. Incident response

Suspected security issues may be reported to support@margynx.com. Privacy-related concerns may be sent to privacy@margynx.com. Internal handling should include triage, containment, remediation, and notifications where required by law or contract.

6. Shared responsibility

Customers are responsible for protecting their own credentials, using strong passwords, assigning access appropriately, and reviewing outputs before taking pricing or operational action.
